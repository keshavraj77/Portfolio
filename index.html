LangGraph Multi-Agent Orchestration Engine
Spearheaded the design and implementation of a LangGraph-based agent orchestration pattern where agents reason and select tools in a single inference step — no separate planning phase. The engine manages isolated virtual conversation threads between a root agent and each of its collaborators, so their contexts never bleed into each other regardless of how many agents are in the graph. The graph supports three agent types in a single unified flow: native agents (standard ReAct), external A2A agents (calling remote agent servers), and flow-based agents (sub-workflows). The engine auto-selects the correct execution path based on the model in use — different inference providers require different tool-calling conventions, and the graph handles that transparently so the calling code never needs to know which backend it's talking to.
LangGraph · Groq · Bedrock · Cerebras · vLLM · Python asyncio
🧠
Progressive-Disclosure Skill Execution System
Agents were hallucinating and picking the wrong tools because their prompts carried the full context of every possible task at once — too much noise for the model to stay focused. The fix: a progressive-disclosure architecture where a compact skill catalog sits in the system prompt for intent classification, and only the selected skill's full instructions and tools are injected on-demand. Two loading modes were built: Inline mode — the skill is injected directly into the current turn's context window; Fork mode — the skill runs inside an isolated sub-graph so it can't affect the root agent's state. The system shipped end-to-end: a CLI (skills import / update / list / export / upload-script), an object-storage-backed script loader (MinIO / IBM COS), and the runtime injection engine inside the orchestration graph. Result: 82% smaller base system prompts, 56% less context per turn, and a measurable drop in tool-selection hallucinations.
82% smaller base prompts · 56% less per-turn context · CLI + Object Storage

A2A Protocol — Client & Server Implementations
Implemented both sides of the Agent-to-Agent (A2A) open protocol — the standard that lets AI agents built on different frameworks communicate with each other.

Client (our platform calls external agents): Built a LangGraph→A2A JSON-RPC adapter with multi-turn correlation tracking so a single conversation can span multiple A2A round trips. Async push-notification callbacks land over Redis Streams so external agents can push updates without the client polling. A 3-tier task-state classifier (deterministic rules → heuristic pattern matching → LLM fallback) minimises latency on the critical path by only escalating to the LLM when rules can't resolve the state.

Server (external systems call our agents): Exposed message/send, message/stream, tasks/get / list / cancel JSON-RPC endpoints with lazy task creation, deterministic SHA-256 artifact IDs for retry-safe idempotency, and dual-signal cancellation (local file marker + Redis pub/sub) that propagates across all pods in under 1 ms.
Dual-signal cancellation <1ms across pods · Redis Streams · SHA-256 artifact IDs

MongoDB → PostgreSQL Migration — Millions of Records, Multiple Data Centers
Led the migration of the entire document and chunk store from MongoDB to PostgreSQL across multiple data centers with no issues reported post-migration.

The hardest part wasn't volume — it was character encoding incompatibility. MongoDB permitted character sequences and byte patterns that PostgreSQL outright rejects at the driver level — years of production data had accumulated edge cases that would silently fail or corrupt on import. Built a multi-stage sanitisation pipeline: encoding detection (charset-normalizer), Unicode normalisation (NFC), and targeted character replacement, handling every edge case reliably at high throughput without data loss.

On the performance side: asyncpg with 6,000-row batch inserts, Python multiprocessing for parallelism across data centers, IBM COS (boto3) for intermediate artifact storage, and exponential-backoff retry for transient failures — making the pipeline resumable and safe to re-run at any point.
asyncpg · 6K rows/batch · multiprocessing · IBM COS · exponential-backoff retry

Interview Prep Agent (A2A Protocol)
An A2A-protocol-compliant multi-turn AI agent built to demonstrate the full spec in a realistic use case. Powered by LangGraph ReAct loop + Gemini 2.0 Flash.

The agent understands your intent conversationally (no keyword matching — natural language parsed by the LLM), then conducts live research to find relevant preparation resources. Each resource is ranked on a 0–10 relevance score with a written explanation, so you know why each one was recommended, not just that it was.

From the ranked resources it synthesises a personalised, day-by-day study plan tailored to your timeline and target role. For long-running research tasks, the agent uses A2A's async push-notification callbacks — results are delivered to a webhook rather than holding the connection open.

Built using the official Google A2A SDK and designed to be callable by any A2A-compatible orchestrator, including AgentComm.
